01 — The Master Build Prompt

Use this: When asking any AI (Claude, ChatGPT, Cursor, GitHub Copilot) to help you build or extend this project. This is the "God prompt" that gives full context. 
You are an expert Chrome Extension developer specializing in Manifest V3, DOM manipulation, and AI API integration.

Build a fully working Google Chrome Extension called "Hallucination Auditor" with the following complete specification:

=== PURPOSE ===
A real-time hallucination and lie detector that watches any conversation on ChatGPT (chatgpt.com), Claude.ai, and Gemini. It analyzes BOTH the user's prompt AND the AI's response, then highlights specific claims that are false or uncertain directly in the chat interface.

=== CORE FEATURES (must implement all) ===

1. DOM OBSERVER: Use MutationObserver to detect when AI finishes streaming a response. Works on chatgpt.com, claude.ai, gemini.google.com.

2. CLAIM HIGHLIGHTER: Inject colored underlines into the AI message bubble:
   - Red underline = hallucination (false claim)  
   - Yellow dashed underline = uncertain claim
   - Do NOT modify verified claims (no green highlight — only underline problematic ones)

3. HOVER TOOLTIP: When user hovers a highlighted claim, show a tooltip with:
   - Verdict (🔴 HALLUCINATION or 🟡 UNCERTAIN)
   - Confidence percentage
   - Brief reason (why it's wrong)
   - Correction (correct information to replace it)
   - Wikipedia snippet (if available from Wikipedia API)

4. TRUST SCORE BADGE: After each AI response, inject a small floating badge showing:
   - Overall trust percentage (0–100%)
   - Animated counter (counts up from 0)
   - Pills showing verified/uncertain/hallucination counts
   - Badge color: green ≥80%, yellow 55–79%, red <55%

5. FRESH START PANEL: When ≥3 hallucinations detected OR trust score <40%, show a red warning button "⚡ Fresh Start". Clicking it:
   - Calls Claude API to summarize the conversation, removing hallucinated content
   - Shows a floating panel with the clean summary as a copy-paste prompt
   - "Open New Chat" button links to chatgpt.com/claude.ai/gemini

6. POPUP UI (3 tabs):
   Tab 1 - Live Audit: Ring chart showing trust score, claim list (non-verified claims only), fresh start button if needed
   Tab 2 - Session Stats: Verified/uncertain/hallucination counts, accuracy bar chart, reset button
   Tab 3 - Settings: API key input, domain mode selector, sensitivity, toggles for auto-analyze/badge/enable

7. OPTIONS PAGE: First-run setup page with step-by-step API key instructions

=== TECHNICAL SPEC ===

Manifest Version: 3 (MV3)
API: Anthropic Claude (claude-sonnet-4-20250514) via https://api.anthropic.com/v1/messages
Secondary API: Wikipedia REST API for claim cross-verification
Storage: chrome.storage.sync for settings/API key, chrome.storage.local for session stats
Message passing: content-script ↔ service-worker ↔ popup via chrome.runtime.sendMessage

File structure:
hallucination-auditor/
├── manifest.json
├── src/background/service-worker.js   (API calls only — no DOM access)
├── src/content/content-script.js       (DOM injection, MutationObserver)
├── src/content/injected.css            (highlight styles, tooltip, badge, panels)
├── src/popup/popup.html + popup.js + popup.css
└── src/options/options.html

=== CLAIM EXTRACTION PROMPT (use this exact system prompt for Claude API) ===

System: "You are a world-class AI hallucination detector. Extract all factual claims from the given AI response and assess each one. Return ONLY valid JSON with no markdown:
{
  claims: [{text, verdict: verified|uncertain|hallucination, confidence: 0-100, reason, correction}],
  trustScore: 0-100,
  overallVerdict: string,
  shouldSuggestFreshStart: boolean,
  topIssue: string|null
}"

=== PLATFORM-SPECIFIC SELECTORS ===

ChatGPT: AI messages = [data-message-author-role="assistant"], content = .markdown
Claude.ai: AI messages = [data-testid="assistant-message"], content = .prose  
Gemini: AI messages = model-response, content = .markdown

Use MutationObserver watching for childList + subtree + attributes (data-is-streaming).
Delay analysis by 800ms after message appears to ensure streaming is complete.

=== DESIGN REQUIREMENTS ===
- Dark theme: background #07070d, surface #0e0e18, accent #00c47d
- Font: system-ui / Inter for UI, JetBrains Mono for scores/code
- All UI is non-intrusive — don't break the platform's layout
- Tooltips are positioned to avoid viewport overflow
- All panels have close buttons and smooth animations

=== IMPORTANT CONSTRAINTS ===
- Service worker handles ALL external API calls (CORS requirement in MV3)
- Content script only does DOM manipulation and sends messages to service worker  
- API key stored in chrome.storage.sync, never hardcoded
- Analyze max 3000 chars of AI response to control API costs
- Only run analysis on platforms in host_permissions list

02 — Service Worker Prompt

Use this when you need to rebuild or debug only the API layer (service-worker.js). 
Write a Chrome Extension Manifest V3 service worker (service-worker.js) that:

1. Handles chrome.runtime.onMessage for these actions:
   - "analyzeText": takes {text, context, domainMode, sensitivity} → calls Anthropic Claude API → returns claim analysis JSON
   - "generateFreshStart": takes {history} → calls Claude API → returns {summary} for clean re-prompt  
   - "getSessionStats": returns stats from chrome.storage.local
   - "resetStats": resets session stats to zero

2. Claude API call (POST https://api.anthropic.com/v1/messages):
   - Model: claude-sonnet-4-20250514
   - Auth header: x-api-key from chrome.storage.sync
   - System prompt: fact-checker that returns ONLY JSON with {claims, trustScore, overallVerdict, shouldSuggestFreshStart, topIssue}
   - Each claim: {text, verdict, confidence, reason, correction}
   - verdicts: "verified" | "uncertain" | "hallucination"

3. Wikipedia verification: for top 2 hallucination claims, call Wikipedia REST API summary endpoint. Attach wikiContext to claim.

4. Update chrome.storage.local sessionStats after each analysis.

5. On chrome.runtime.onInstalled: set default settings, open options page.

Return only the complete service-worker.js file with no explanation.

03 — Content Script Prompt

Use this to rebuild content-script.js — the DOM injection engine. 
Write a Chrome Extension content script (content-script.js) that injects into chatgpt.com, claude.ai, and gemini.google.com and does the following:

PLATFORM DETECTION:
- Auto-detect platform from window.location.hostname
- Use different CSS selectors per platform:
  ChatGPT: aiMsg=[data-message-author-role="assistant"], content=.markdown
  Claude:  aiMsg=[data-testid="assistant-message"], content=.prose
  Gemini:  aiMsg=model-response, content=.markdown

MUTATION OBSERVER:
- Watch document root for new AI message elements
- Also watch for data-is-streaming attribute changes (Claude)
- Delay 800ms before triggering analysis to ensure streaming complete
- Track analyzed message IDs in a Set to avoid re-analyzing

ANALYSIS PIPELINE (async):
1. Extract innerText from AI message element
2. Build conversation context from last 6 messages
3. Send to service worker: {action:"analyzeText", text, context, domainMode, sensitivity}
4. On result: inject highlights, badge, and (if needed) fresh start button

HIGHLIGHT INJECTION:
- Use TreeWalker (NodeFilter.SHOW_TEXT) to find text nodes containing the claim
- Split text node into [before, span, after] using insertBefore/removeChild
- Add span.className = "ha-claim ha-{verdict}" with dataset for tooltip data
- Attach mouseenter/mouseleave listeners for tooltip

TRUST SCORE BADGE:
- Create .ha-badge element and insert after message element
- Show animated counter (setInterval) counting from 0 to score
- Color-code based on score: ≥80 green, ≥55 yellow, <55 red

FRESH START BUTTON:
- Show .ha-fresh-btn when shouldSuggestFreshStart OR hallucinationTotal >= 3
- On click: call service worker generateFreshStart → show floating .ha-fs-panel
- Panel has: copy-to-clipboard, open new chat, close button

CONVERSATION TRACKING:
- Track user messages in a conversationLog array: [{role, text}]
- Keep last 20 messages for context building

MESSAGE LISTENER:
- Handle "getConversationStats", "settingsUpdated", "triggerFreshStart" from popup

All UI elements use class names prefixed with "ha-" to avoid conflicts with host page CSS.
Return only the complete content-script.js.
